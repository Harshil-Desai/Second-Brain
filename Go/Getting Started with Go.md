# Getting Started with Go

## Why Go?

Go has **goroutines** — lightweight primitives that let you run multiple tasks across threads simultaneously, with built-in concurrency support. That makes Go a strong fit for performant applications running on scaled and distributed systems.

## Where it's used

- Server side (backend)
- Fast build time and startup time
- Fewer resources required
- Compiled language — produces a single binary that runs across operating systems

## Notable software written in Go

- Docker
- Kubernetes
- CockroachDB
- HashiCorp Vault
